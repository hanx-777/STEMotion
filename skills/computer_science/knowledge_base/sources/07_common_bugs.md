# 程序设计常见 Bug

## 数组越界

访问数组时下标超出合法范围（C++ 中是未定义行为）。常见原因：循环条件写成 `<=` 而非 `<`，忘记数组从 0 开始。

## 空指针解引用

对空指针（`nullptr`）进行解引用操作。常见原因：忘记检查指针是否为空，链表遍历结束后继续访问。

## 整数溢出

算术运算结果超出整数类型的表示范围。常见场景：两个大整数相乘、累加求和。解决方案：使用更大的类型（`long long`）或在运算前检查。

## 死循环

循环条件永远为真。常见原因：循环变量未更新、条件写错、浮点数精度导致条件永远不满足。

## 栈溢出

递归深度过大，调用栈空间耗尽。常见原因：忘记基准情况、递归深度与输入规模成正比。

## 内存泄漏

动态分配的内存未释放。在使用 `new` 后忘记 `delete`。现代 C++ 建议使用智能指针（`unique_ptr`、`shared_ptr`）。

## 运算符优先级错误

`a & b == c` 被解析为 `a & (b == c)` 而非 `(a & b) == c`。解决方案：多加括号。

## 浮点数比较

直接用 `==` 比较浮点数可能因精度问题失败。应使用容差比较：`abs(a - b) < epsilon`。
