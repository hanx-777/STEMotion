# 表达式求值

## 中缀表达式

中缀表达式是运算符写在操作数之间的常见形式，如 `3 + 4 * 2`。需要考虑运算符优先级和括号。

## 中缀转后缀（逆波兰表达式）

使用调度场算法（Shunting Yard Algorithm）：
1. 初始化运算符栈和输出队列。
2. 从左到右扫描中缀表达式。
3. 遇到操作数：直接输出。
4. 遇到运算符：弹出栈中优先级不低于当前运算符的所有运算符并输出，然后将当前运算符压栈。
5. 遇到左括号：压栈。
6. 遇到右括号：弹出并输出直到遇到左括号（左括号弹出但不输出）。
7. 扫描结束后，弹出栈中所有运算符并输出。

## 后缀表达式求值

使用栈：
1. 从左到右扫描后缀表达式。
2. 遇到操作数：压栈。
3. 遇到运算符：弹出两个操作数，计算结果，将结果压栈。
4. 扫描结束后，栈中唯一元素即为结果。

时间复杂度：\(O(n)\)。

## 括号匹配

使用栈判断括号是否匹配：
1. 遇到左括号：压栈。
2. 遇到右括号：若栈为空或栈顶不匹配则失败，否则弹出栈顶。
3. 扫描结束后栈应为空。

需要考虑的边界情况：空字符串、只有左括号、只有右括号、嵌套括号、不同种类括号混合。

## 常见误区

- 在后缀求值时弹出操作数的顺序搞反（先弹出的是右操作数）。
- 忘记处理除法的整数截断问题。
- 在括号匹配中只检查数量不检查类型。
