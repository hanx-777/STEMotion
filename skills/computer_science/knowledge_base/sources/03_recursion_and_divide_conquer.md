# 递归与分治

## 递归

递归函数是直接或间接调用自身的函数。递归三要素：
1. 基准情况（base case）：不再递归的终止条件。
2. 递归关系（recursive case）：将问题分解为更小的子问题。
3. 进展保证：每次递归调用必须向基准情况靠近。

示例——阶乘：
```cpp
int factorial(int n) {
    if (n <= 1) return 1;        // 基准情况
    return n * factorial(n - 1); // 递归关系
}
```

## 分治法

分治法将问题分解为若干规模较小的子问题，递归求解，然后合并结果。
1. 分解（Divide）：将问题分为子问题。
2. 解决（Conquer）：递归求解子问题。
3. 合并（Combine）：将子问题的解合并为原问题的解。

经典应用：归并排序（\(O(n\log n)\)）、快速排序（平均 \(O(n\log n)\)）、二分查找（\(O(\log n)\)）。

## 递归转迭代

任何递归都可以用栈手动模拟调用栈转为迭代。尾递归可以直接转为循环。

## 常见误区

- 忘记写基准情况导致无限递归。
- 递归中重复计算导致指数级时间复杂度（如朴素斐波那契）。
- 栈溢出：递归深度过大时，调用栈空间不足。
