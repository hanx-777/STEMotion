# 算法复杂度

## 时间复杂度

时间复杂度描述算法执行时间随输入规模增长的趋势。使用大 O 记号：\(O(f(n))\) 表示上界。

常见复杂度（从小到大）：
- \(O(1)\)：常数时间，如数组随机访问。
- \(O(\log n)\)：对数时间，如二分查找。
- \(O(n)\)：线性时间，如遍历数组。
- \(O(n\log n)\)：如归并排序、快速排序（平均）。
- \(O(n^2)\)：如冒泡排序、选择排序。
- \(O(2^n)\)：如穷举所有子集。
- \(O(n!)\)：如穷举所有排列。

## 空间复杂度

空间复杂度描述算法使用额外空间随输入规模增长的趋势。

常见空间复杂度：
- \(O(1)\)：原地算法，如原地排序。
- \(O(n)\)：需要与输入等规模的额外空间，如归并排序。
- \(O(\log n)\)：递归栈深度，如快速排序（平均）。

## 分析方法

循环分析：嵌套循环相乘，顺序循环取最大值。
递归分析：主定理（Master Theorem）或递归树法。
均摊分析：对一系列操作取平均，如动态数组的 push 操作。

## 最好、最坏、平均情况

最好情况：输入恰好使算法最快完成（如已排序数组的插入排序）。
最坏情况：输入使算法最慢完成（通常更有意义）。
平均情况：对所有可能输入取平均（通常难以精确计算）。

## 常见误区

- 混淆 \(O\)（上界）和 \(\Theta\)（紧确界）。
- 忽略常数因子和低阶项在实际性能中的影响。
- 将"最坏情况"等同于"总是这么慢"。
