# 参赛效果验证测试案例

本目录用于 XH202620 参赛效果验证，记录典型输入、预期输出结构、人工检查点和后续真实用户反馈材料。

注意：本目录中的用户反馈和效果报告当前为模板或测试设计，不包含伪造的真实用户反馈。

## Case 01：非零高度斜抛运动建模

- 任务类型：`step_solution`
- 测试目标：验证系统能基于大学物理力学知识库完成非零初始高度条件下的斜抛运动建模、分步推导、参数分析、引用来源和可视化参数生成。
- 输入问题：在大学物理力学实验中，某小球从高度 h = 1.20 m 的平台边缘以初速度 v0 = 8.0 m/s、仰角 θ = 35° 斜向上抛出。忽略空气阻力，取 g = 9.8 m/s²。请建立二维运动模型，求小球落地时间、水平射程、最大高度，并分析 θ 对射程的影响。要求说明模型假设、公式适用条件、单位检查和易错点，并给出可视化参数。
- 预期输出结构：模型假设、已知量提取、物理模型判断、分步推导、计算结果、参数分析、易错点、引用来源、可视化参数。
- 验证方式：与人工标准答案对比，检查非零高度条件下的落地时间求解（二次方程）、最大高度公式、射程公式、参数敏感性分析、引用来源和 `projectile_motion` visualization hint。

详见 [physics_case_01_projectile_motion.md](physics_case_01_projectile_motion.md)。

## Case 02：向心加速度的矢量分析

- 任务类型：`knowledge_qa`
- 测试目标：验证系统能从矢量微分角度解释向心加速度的物理意义，说明速度矢量方向变化与加速度的关系。
- 输入问题：在大学物理中，为什么匀速圆周运动的速度大小不变但仍需引入向心加速度？请从速度矢量的微分定义出发，说明加速度的物理意义、方向特征和公式推导过程。
- 预期输出结构：核心概念、矢量推导、方向特征、学习建议、引用来源。
- 验证方式：检查是否从 a = dv/dt 出发推导，公式 `a = v²/r` 的推导过程，加速度方向指向圆心，引用来源。

详见 [physics_case_02_circular_motion.md](physics_case_02_circular_motion.md)。

## Case 03：斜抛运动公式误用诊断

- 任务类型：`misconception_diagnosis`
- 测试目标：验证系统能识别学生答案中的公式适用条件误用和概念错误，给出正确的推导过程和复习建议。
- 输入问题：学生在求解斜抛运动最大高度时写道：H = v0² / (2g)，并认为发射角越大射程一定越大。请判断该答案是否正确，分析其中的公式误用和概念错误。
- 预期输出结构：学生可能误解、错误原因、射程误区、纠正思路、巩固练习、引用来源。
- 验证方式：检查是否指出 H = v0²/(2g) 仅适用于竖直上抛，正确公式应为 H = (v0 sinθ)²/(2g)，射程与角度的关系分析。

详见 [physics_case_03_misconception.md](physics_case_03_misconception.md)。

## Case 04：非零高度斜抛运动课堂演示

- 任务类型：`teacher_prep`
- 测试目标：验证系统能生成面向大学物理课堂的完整演示方案，包含模型假设、参数变化实验和学生易错点。
- 输入问题：请为大学物理"非零初始高度条件下的斜抛运动建模"设计一段 15 分钟课堂演示，要求包含教学目标、模型假设、公式推导、参数变化实验、课堂互动问题、学生易错点和课后拓展练习。
- 预期输出结构：教学目标、模型假设、公式推导、参数变化实验、互动问题、学生易错点、课后拓展、引用来源。
- 验证方式：由教师检查是否适合大学物理课堂，是否包含非零高度条件的特殊分析，是否能突出助教备课场景。

目的：突出系统不只是学生助学，也能服务教师助教场景。

## 大学层次默认问题设计原则

1. **不只考查代公式**：问题应包含建模过程、假设说明和参数分析，而不是简单的数值代入。
2. **包含模型假设**：明确要求说明物理模型的适用条件和简化假设（如忽略空气阻力、质点模型等）。
3. **包含参数分析**：要求分析参数变化对结果的影响（如发射角对射程的影响、稀释对电离度的影响）。
4. **包含单位或条件检查**：要求验证公式适用条件、近似条件是否成立、单位一致性。
5. **能触发 RAG 引用和结构化回答**：问题应足够具体，能从知识库中检索到相关依据并生成分步结构化回答。
6. **可视化类问题应能触发 visualization_hint 或 deep-interaction 生成**：包含可提取的物理参数（如 v0、θ、g）。
