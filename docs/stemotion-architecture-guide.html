01 · Overview

架构总览

STEMotion 采用 Next.js 模块化单体架构，将代码分为六层：页面层、功能层、组件层、库层、平台层和共享层。 核心业务逻辑集中在 src/lib/（三条 AI 多智能体管线）和 src/features/（垂直切片）， src/app/ 只负责路由和页面组装。

架构核心原则：src/app/ 只做路由和页面组装，不放业务逻辑； src/features/ 按垂直切片组织，每个 feature 有独立的 application/（服务）和 ui/（工作台组件）； src/lib/ 是最重的一层，包含三条 AI 多智能体管线和 13 个 Zustand 状态管理。

03 · Feature Layer

Layer 2 · 功能层 src/features/

功能层采用垂直切片设计，每个 feature 有独立的 application/（服务逻辑）和 ui/（工作台组件）。 页面层只负责加载对应的 workbench，不直接处理业务。

01 rag — RAG 核心
src/features/rag/

系统最核心的 feature。包含 RAG 问答服务（application/ragAskService.ts）、客户端调用（client/ragClient.ts）、请求/响应契约（contracts.ts）、会话状态管理（state/ragSessionStore.ts）、错误归因（state/ragAskErrorAttribution.ts）、可视化流程（state/visualizationFlow.ts），以及学生工作台（ui/RagWorkbench.tsx）和学科控制台（ui/SubjectRagConsole.tsx）。

02 deep-interaction — 深度交互
src/features/deep-interaction/

承载 Guided Planning → LearningBlueprint → 多 Agent 评审 → 生成的完整管线。包含深度交互服务（application/deepInteractionService.ts）和两个 UI 入口：深度交互工作台（ui/DeepInteractionWorkbench.tsx）和可视化实验页面（ui/LabSurfacePage.tsx）。

03 subjects — 学科管理
src/features/subjects/

管理学科 Skill 的加载、切换和配置。核心服务：application/subjectService.ts，底层依赖 src/lib/subjects/subject_manager.ts。

04 settings — 模型配置
src/features/settings/

管理 OpenAI / Claude 模型 profile 的 CRUD 和切换。服务层：application/modelProfileService.ts，UI：ui/SettingsWorkbench.tsx。

05 knowledge — 知识库健康度
src/features/knowledge/

展示各学科知识库的索引状态、源文件数、chunk 数和覆盖范围。核心文件：knowledgeHealth.ts。

06 assets — 教学资产管理
src/features/assets/

管理本地交互库的 artifact。UI：ui/AssetsWorkbench.tsx，底层依赖 src/lib/stores/artifactStore。

07 rag-lab-bridge — RAG → Lab 桥接
src/features/rag-lab-bridge/

将 RAG 输出桥接到 Lab 实验生成。核心文件：buildLabPrompt.ts，将 RAG 的结构化回答转换为深度交互的实验 prompt。

06-07 · Platform & Shared

Layer 5-6 · 平台层与共享层

这两层非常薄，只放跨功能共享的基础设施代码。

平台层 src/platform/

文件职责
api/http.tsHTTP 客户端封装，统一请求/响应处理
errors.ts错误类型定义，统一错误处理契约

共享层 src/shared/

文件职责
api/types.ts跨功能共享的 API 类型定义
为什么这么薄？STEMotion 的业务逻辑集中在 src/lib/（AI 管线）和 src/features/（垂直切片）， 平台层和共享层只承载真正需要跨 feature 共享的基础设施。这种设计避免了"共享层膨胀"的问题。

09 · State Management

状态管理全景

系统使用 Zustand 作为状态管理方案，共 13 个 store，覆盖实验、交互、回放、RAG 会话、生成进度等维度。

Store职责关联页面
experimentStore当前实验配置和状态/, /lab
interactionSessionStore交互会话管理/lab, /deep-interaction
interactionPersistence交互持久化（localStorage）/assets
artifactStoreartifact 管理/assets
playbackStore回放控制/player
ragSessionStoreRAG 会话状态/learn, /teach
generationProgressStore生成进度跟踪/lab, /deep-interaction
deepInteractionUIStore深度交互 UI 状态/lab, /deep-interaction
deepWidgetIframeStore深度 Widget iframe 状态/lab
widgetIframeStore标准 Widget iframe 状态/
assistantStore助手面板状态/, /lab
researchLogStore研究日志记录全局
toastStoreToast 通知全局
uiStore全局 UI 状态（侧边栏等）全局
持久化策略：interactionPersistence 使用 Zustand persist 中间件将 artifact 数据写入 localStorage， 其余 store 为运行时内存状态。页面刷新后，persist 的数据自动恢复。
