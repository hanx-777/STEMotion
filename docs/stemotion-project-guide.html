01 · Executive Summary

执行摘要

STEMotion Physics Skill 是一个面向高校大学物理力学课程的可追溯 RAG 智能教学系统。 系统以本地知识库检索为优先，结合大模型生成能力，为学生提供分步解题、错因诊断，为教师提供备课支持和课堂演示。

核心 Takeaways

  • 本地知识库优先：系统先从本地课程知识库检索依据，再由大模型生成回答，确保引用可追溯
  • 四学科覆盖：大学物理力学（深度落地）、高等数学、大学化学、程序设计与数据结构（轻量配置）
  • 38 个测试文件：覆盖 RAG 管线、API 契约、可视化、架构边界等维度
  • 版本化 API：/api/v1/* 为正式接口，/api/* 保留兼容

02 · Architecture

系统架构

采用 Next.js 16 模块化单体架构，前端页面、Route Handlers、业务逻辑和平台能力分层清晰。

分层结构

层级 目录 职责
页面层 src/app/ Next.js App Router 页面和 Route Handler
功能层 src/features/ 领域功能模块：rag、subjects、settings、deep-interaction、knowledge、assets、rag-lab-bridge
平台层 src/platform/ HTTP 处理、错误处理等平台能力
共享层 src/shared/ 跨功能共享类型和工具
组件层 src/components/ 共享 UI 组件
库层 src/lib/ RAG 检索、学科管理、配置等底层库

数据流

用户请求 → 前端页面 → /api/v1/rag/ask → SubjectManager 加载 Skill → Retriever 检索本地知识库 → 可选 WebSearchProvider 补充检索 → 构建 citations 和 prompt → OpenAI / Claude 生成最终回答 → 返回结构化回答 + 引用来源 + 可视化提示

关键设计原则

  • RAG 只负责检索依据、组织上下文和标注来源，最终回答由大模型生成
  • 本地课程资料是优先可信来源，网络检索资料只作为补充
  • 无可靠来源时仍调用大模型，但必须明确说明依据不足，不生成虚假 citations
  • 版本化 API（/api/v1/*）为正式接口，旧版（/api/*）在一个版本周期内作为兼容入口
03 · Core Features

核心功能

系统围绕"学生学习"和"教师教学"两个主场景，提供从知识检索到可视化实验的完整教学闭环。

学生学习

知识问答

针对物理概念、公式含义、学习建议等开放性问题，系统从本地知识库检索相关章节，结合大模型生成结构化解释，附带可追溯引用来源。

分步解题

接收题目后，系统自动提取已知量、判断物理模型、分步推导、给出结果和易错点，并返回 visualization_hint 供前端渲染运动轨迹。

错因诊断

学生提交错误答案后，系统分析错误原因、给出纠正思路和巩固练习，帮助学生理解"为什么错"而非仅仅"对不对"。

教师教学

教师备课

生成教学目标、课堂导入、板书推导和互动提问，帮助教师快速准备课堂教学材料。

可视化实验

支持斜抛轨迹、圆周运动、机械能守恒、动量碰撞、简谐振动等物理过程的交互式可视化。教师可在课堂上演示参数变化对运动轨迹的影响。

知识库管理

学科 Skill 切换

每个学科拥有独立的配置文件（skill.yaml）、系统提示词、回答模板和知识库。支持一键切换学科，检索自动隔离。

知识库健康度

/knowledge 页面可视化展示各学科知识库的索引状态、源文件数、chunk 数和覆盖范围，便于运维监控。

教学资产管理

/assets 页面管理本地交互库，支持保存、打开、筛选和删除 artifact，方便教师复用教学素材。

05 · Getting Started

快速开始

五个步骤即可启动系统：安装依赖、配置模型、构建索引、启动服务、验证功能。

1. 安装依赖

npm install

2. 配置模型

复制示例配置并填入 API Key：

cp model-profiles.example.json model-profiles.json

支持 OpenAI 和 Claude 两种模型接口，可在 /settings 页面可视化配置。

3. 构建知识库索引

npm run rag:ingest -- --subject all

索引产物写入 .stemotion/vector-store/，该目录是本地运行态，不提交到 Git。

4. 启动开发服务器

npm run dev

默认地址：http://localhost:3001

5. 命令行测试

npm run rag:query -- --subject physics_mechanics --task step_solution --question "斜抛运动最大高度公式是什么"

常用脚本

07 · RAG Workflow

RAG 工作流

RAG（Retrieval-Augmented Generation）是系统的核心能力，负责从本地知识库检索依据，结合大模型生成可追溯的教学回答。

工作流程

用户提问 → SubjectManager 验证学科并加载 Skill → Retriever 检索本地知识库 chunks → 若本地得分低于阈值且启用网络检索 → WebSearchProvider 补充检索 → 构建 citations 和 prompt → OpenAI / Claude 生成最终回答 → 返回 answer + sections + citations + visualization_hint

任务类型

task_type 中文名 用途
knowledge_qa 知识问答 概念解释、公式含义、学习建议
step_solution 分步解题 已知量提取、模型判断、推导、结果和易错点
misconception_diagnosis 错因诊断 分析学生错误答案、纠正思路和巩固练习
teacher_prep 教师备课 教学目标、课堂导入、板书推导和互动提问

引用追溯机制

  • 返回 citations、retrieved_chunks、source_summary
  • 严格区分本地课程资料和网络补充资料
  • 无可靠来源时明确标注"当前知识库和网络检索中未找到可靠依据"
  • 不编造文献、页码或课程来源

网络检索配置

变量 说明
STEMOTION_WEB_SEARCH_PROVIDER=mock 使用 MockWebSearchProvider，适合测试和演示
STEMOTION_WEB_SEARCH_PROVIDER=custom-json 使用自定义 JSON 搜索接口
STEMOTION_WEB_SEARCH_ENDPOINT 自定义搜索 API 地址
STEMOTION_WEB_SEARCH_API_KEY 自定义搜索 API Key

  top_k: 4

  - 引用来源

添加新学科

  1. 复制任一现有学科目录
  2. 修改 skill.yaml、system_prompt.md、answer_template.md
  3. 将文档放入对应 knowledge_base/sources/
  4. 运行 npm run rag:ingest -- --subject your_subject_name
09 · API Reference

API 接口

系统提供版本化 REST API，/api/v1/* 为正式接口，旧版 /api/* 保留兼容。

学科与 RAG

方法 路径 说明
GET /api/v1/subjects 获取可用学科 Skill、知识库状态、工具和回答规范
GET /api/v1/subjects/default 获取默认学科
PATCH /api/v1/subjects/default 设置默认学科
POST /api/v1/rag/ask 执行学科 RAG 问答，返回结构化回答、引用来源、证据包、复核报告和可视化提示

模型配置

方法 路径 说明
GET /api/v1/model-profiles 读取脱敏后的模型 profile 摘要
POST /api/v1/model-profiles 新建或更新模型 profile
PATCH /api/v1/model-profiles 切换当前 active profile
DELETE /api/v1/model-profiles/{id} 删除模型 profile
POST /api/v1/model-profiles/models 获取 OpenAI 或 Claude 模型列表

深度交互生成

方法 路径 说明
POST /api/generate 标准实验室一次性生成
POST /api/v1/deep-interaction/planning 深度交互生成前的澄清与教师计划
POST /api/v1/deep-interaction/generate 深度交互 SSE 生成
POST /api/v1/deep-interaction/follow-up 基于当前 HTML 的追问修改

重点验收场景

  1. /settings 可以新增 OpenAI / Claude profile，获取模型列表并切换 active profile
  2. /learn 默认学科为大学物理力学，旧路由 /student、/rag 会重定向到 /learn
  3. 斜抛运动问题能返回结构化分步解题、引用来源和 visualization_hint
  4. 无可靠来源时，仍由模型回答，但明确提示依据不足，不伪造 citations
  5. 切换到其他学科时，检索不会混用其他学科知识库
  6. 模型配置错误时，RAG API 返回明确错误，前端不展示检索片段拼接成的假回答
  7. 典型案例 demo fallback 只用于演示兜底，并明确标注"演示样例结果"
